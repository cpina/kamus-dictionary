What is this?

The first and most important thing: currently this is a personal toy project. Do not rely on Kamus. It might have downtimes, it might disappear, it's structure may change. On the other hand, I'm using it daily, so I plan to keep maintaining it.

Before I started, I had two things in mind:

  • How could I make translation data from Wiktionary easier to access?
  • I wanted to learn more about the structure of Wiktionary and the project itself

Kamus dictionary tries to help a user to find translations from Wiktionary. It reads the relevant pages and presents translation information in an easier way, filtering out what's not needed for the translations.

As an example, try to find the translation from the English word "table" to Catalan, using the Wiktionary page or the Kamus page.

I hope that Kamus might help some users to use Wiktionary and hopefully, improve it.

More languages are coming

Currently, Kamus helps a user translate from English to any language listed. Most likely, the next languages I will add, will be Catalan and Spanish. This will allow translations from these languages, to any of the other languages listed. I have some ideas for how to implement Kamus to translate "from any to any" instead of having to implement languages manually, but I'm still evaluating it.

Quick "good to know"

When loading a translation, the data is fetched from Wiktionary in real time. Therefore, if you edit Wiktionary and reload Kamus you will see the improvements in real time.

The list of words for the autocompletion is fetched regularly (about once a week). New entries on Wiktionary will take some time to appear in the autocompletion list of Kamus.

JavaScript and Cookies

JavaScript is currently required for the "word" to translate autocompletion.

Cookies are used in order to save the "From" and "To" languages.
